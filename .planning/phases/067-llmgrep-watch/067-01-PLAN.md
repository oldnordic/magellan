- Watch subcommand in CLI with query, path, and mode flags
- Signal handling for clean shutdown (SIGINT/SIGTERM)

**Module structure:**
```rust
//! Watch command implementation for real-time query results

use anyhow::Result;
use std::path::PathBuf;
use std::sync::atomic::{AtomicBool, Ordering};
use std::sync::Arc;
use std::sync::mpsc::Receiver;
use std::time::Duration;

use crate::backend::Backend;
use crate::error::LlmError;
use crate::output::{OutputFormat, SearchResponse};
use crate::query::SearchOptions;

#[cfg(feature = "native-v2")]
use sqlitegraph::backend::{PubSubEvent, SubscriptionFilter};
```

**Key functions to implement:**

1. `run_watch()` - Main entry point that:
   - Detects backend format using `Backend::detect_and_open()`
   - Calls `run_watch_with_pubsub()` for native-v2
   - Calls `run_watch_with_filesystem()` for SQLite (with warning)
   - Returns `Result<()>` with `LlmError` for failures

2. `run_watch_with_pubsub()` - Native-v2 implementation that:
   - Subscribes to `SubscriptionFilter::all()` via `backend.subscribe()`
   - Runs initial query and displays results
   - Enters event loop with `rx.recv_timeout(Duration::from_millis(100))`
   - On each event, re-runs query and calls `format_delta()`
   - Handles `Disconnected` error by exiting with message

3. `format_delta()` - Delta display that:
   - Takes `previous: &[SymbolMatch]` and `current: &[SymbolMatch]`
   - Computes added: items in current but not previous
   - Computes removed: items in previous but not current
   - Prints "Added: N, Removed: M" header
   - Lists added items with green "+" prefix (if terminal supports)
   - Lists removed items with red "-" prefix (if terminal supports)

**Pattern reference:** Use `/home/feanor/Projects/magellan/src/watcher/pubsub_receiver.rs:184-214` for event loop structure.

**Signal handling:**
- Accept `shutdown: Arc<AtomicBool>` parameter
- Check `!shutdown.load(Ordering::Relaxed)` in loop condition
- Use `recv_timeout()` to allow checking shutdown flag

**Error handling:**
- Return `LlmError::RequiresNativeV2Backend` if trying to use pub/sub on SQLite
- Return `LlmError::BackendDetectionFailed` if format detection fails
- Log `eprintln!()` for transient errors but continue loop

**DO NOT:**
- Use blocking `recv()` without timeout (can't check shutdown flag)
- Create new subscription on each event (create once, reuse)
- Display full results on every change (use delta mode)
- Assume pub/sub is available on SQLite backend
